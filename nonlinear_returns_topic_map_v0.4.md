# 《非线性回报》选题骨架 v0.4

> 这一版做了两件事:把整档节目重组成「一次模型训练 run」的叙事主线,并把 v0.3 的 77 个选题重筛、合并、砍重复,收敛成约 36 集。
> 主理人背景:AI 算法工程师(机器学习 / 深度学习 / Agent)+ 多年金融行业。这是节目最大的护城河——别人讲人生哲理,你能讲它背后真实的优化机制。
> 形式:一个人的独白 + 讲故事。每集靠一个**具体场景**进入,而不是靠一句金句收尾。

---

## 0. 这版的总思路(先看这一段)

「非线性回报」+「随机梯度下降」其实是同一种世界观:**你不是在解一道有标准答案的题,而是在一片非凸的损失地形里,靠带噪声的小步,一步步往下走。**

走得动的前提有三个,正好对应你的两套专业直觉:

- **凸性 / 非线性 payoff**(金工):有些下注错了归零,有些错了只亏权利金。非线性回报来自后者。
- **优化过程**(ML):损失地形、梯度、学习率、噪声、过拟合、收敛——这些不是比喻,是机制。
- **不归零**(两者共有):一次 NaN 能让整个 run 作废;一次 ruin 能让前面所有复利清零。活下来,是收敛的前提。

所以这一版把节目排成**一次训练的全过程**:从"你在优化什么"开始,到"让模型上线被看见"结束。每个模块是训练的一个阶段。

**关于"不做成金句合集":** v0.3 里 77 个「一句话抓手」有 40+ 个是同一个句式("不是 X,而是 Y")。这一版刻意把每集的核心字段从「抓手」换成「故事入口 + 机制 + 落点」。抓手保留为可选,且要求句式多样化——能用一个画面、一个数据、一个问题代替的,就不要用格言。

- **落点**:它如何对应职业 / 金钱 / 认知里的一个真实选择。
- **来源**:由 v0.3 的哪些选题合并而来。

# 序章 · 世界观

## 序.1 为什么这档节目叫「非线性回报」
- **故事入口**:你给节目取名时的真实考虑——它既指期权那种不对称 payoff,也指随机梯度下降那种"带噪声地往下走"的手感。
- **机制**:线性回报 = 出售时间,封顶;非线性回报 = 凸性,一次行动在未来反复结算。配一张 payoff 图,从你本行讲清楚。
- **落点**:整档节目的世界观锚点。听众听完知道:这不是又一个讲道理的节目,是一个工程师在拆"什么样的努力会被放大"。
- **来源**:B02。

## 序.2 AI 让什么变便宜,让什么变贵
- **故事入口**:作为 AI 工程师,你亲眼看到生成、初稿、代码、调研的成本塌到接近零;同时,定义目标、验收、判断"哪个对"反而越来越贵。
- **机制**:成本结构变化——执行降价,判断升值。这是后面所有模块的前提,所以放在序章而不是单独成集。
- **落点**:为什么"现在"要重新理解非线性回报。
- **来源**:A01、A02、C01(三题合并为节目前提)。

# M1 · 目标函数:你到底在优化什么

> 训练第一步不是跑,是定义 loss。loss 错了,梯度再准、跑得再快,都在往错的方向收敛。

## 1.1 你在最小化哪个 loss
- **故事入口**:一个为了"准确率"优化的模型,在不平衡数据上准确率 99%,却把所有正样本都漏了;或一个最大化 PnL 的策略,把尾部风险全吃进去了。
- **机制**:你优化什么,就得到什么(optimize what you measure)。目标定义错了,越努力越偏。验收标准 = loss 的具体算法;选择质量 = 选目标比执行更影响结果。
- **落点**:很多事不是执行失败,是一开始的目标就没定义清楚。
- **来源**:C03,并入 A04(思考即定义目标)、C04(验收标准)、G08(选择质量 = 选超参)。

## 1.2 好问题比好答案值钱
- **故事入口**:做 Agent / LLM 时,prompt 和目标设定决定了能力上限——同一个模型,问题问得好不好,结果差一个数量级。
- **机制**:问题的质量 = 目标函数的质量,决定了系统能达到的天花板。
- **落点**:AI 越强,会定义问题的人越值钱。
- **来源**:C02。

## 1.3 财务自由是约束条件,不是目标函数
- **故事入口**:组合优化里,你很少"最大化收益",而是"在回撤不超过 X 的约束下"最大化收益。把这个搬到人生上。
- **机制**:钱是约束(constraint),不是要最大化的目标;现金流托底 = 让你不被迫在差选项里选。这样讲钱,避开了财商节目的俗套。
- **落点**:普通人谈钱不庸俗——很多自由都需要现金流当约束托底。
- **来源**:G05,并入 G02(现实感)、G04(正当谋利,简述)。

# M2 · 损失地形:这世界没有标准答案

> 真实世界是非凸的:没有闭式解,只有局部最优、鞍点、平台期,和不断往下走。

## 2.1 人生是非凸优化,不是解方程
- **故事入口**:非凸损失地形没有"标准答案",你只能持续提高"走到更低点"的概率。
- **机制**:非凸 = 无闭式解;你做的是提高好结果的概率,不是求确定解。
- **落点**:把"人生不是解题,而是提高概率"从口号变成一张地形图。
- **来源**:H01。

## 2.2 局部最优:舒适区是一个你下不去的坑
- **故事入口**:训练卡在 local minimum,loss 不降了;要跳出去,得先短暂地"变差"(爬出坑)。对应一份稳定但封顶的工作。
- **机制**:局部最优 vs 全局;线性劳动(干一份活拿一份钱)是一个舒服但出不去的坑。
- **落点**:最可怕的不是辛苦,是你的辛苦困在一个下不去的局部最优里。
- **来源**:B01,并入 B04(你在为谁干活)。

## 2.3 热点、趋势、周期:在地形上分辨噪声和坡度
- **故事入口**:回测里你最怕把噪声当信号。生活里同理:热点是噪声,趋势是真实的坡度,周期是来回震荡。
- **机制**:噪声(情绪/注意力)vs 梯度方向(成本结构/技术结构)vs 振荡(供需/人性)。
- **落点**:怎么判断一个东西是短期热闹还是长期变化。
- **来源**:A03。

# M3 · 数据与信号:你喂给自己什么

> Garbage in, garbage out。你的判断质量,上限取决于你喂给自己的数据。

## 3.1 信息不值钱,注意力才是稀缺算力
- **故事入口**:训练有算力预算,你不可能对所有数据都 backprop。人的注意力就是那块 GPU,信息越便宜,它越贵。
- **机制**:compute / attention budget 有限;信息过载下,能持续停在一个重要问题上的注意力才是稀缺资源。
- **落点**:为什么 AI 时代要保护注意力,而不是消费它。
- **来源**:J06。

## 3.2 读书是更换你的数据分布
- **故事入口**:模型泛化好不好,取决于训练分布。读书 = 主动改变你的输入分布,让你能处理没见过的问题。AI 能"压缩"信息(给你摘要),但替不了你改变分布。
- **机制**:training distribution / distribution shift;阅读过程本身(注意力、结构感、耐心)是不可被摘要替代的训练。
- **落点**:AI 时代要更读书,不是更少——AI 压缩信息,但不能替你形成判断。
- **来源**:J04 + J05 合并。

## 3.3 Data leakage:为什么「看起来对」最危险
- **故事入口**:一个特征里偷偷混进了未来信息,回测漂亮得不真实,上线就崩。这是你本行里最经典的坑。
- **机制**:数据泄漏 ≠ 过拟合(那是 M7)。这里讲"被污染的数据/来源缺失/结构清晰但事实错误",AI 幻觉也是同一类:包装得很像正确的错误。
- **落点**:AI 时代最可怕的不是错误,是看起来对的错误。
- **来源**:C05。

# M4 · 梯度:没有反馈就没有学习

> 你只能在有梯度的地方改进。没有反馈信号,再多 epoch 也学不到东西。

## 4.1 迷茫不是想太少,是没有梯度
- **故事入口**:一个 loss 曲线压根不动的训练 run——不是模型笨,是梯度为零。迷茫是同一回事:不是缺思考,是缺反馈。
- **机制**:零梯度 = 不更新。执行力不靠热血,靠"小动作→拿到反馈→再动"的循环;不确定时先用低成本行动买梯度信息。
- **落点**:破迷茫的办法不是想更久,是制造一个能拿到反馈的小动作。
- **来源**:E07。

## 4.2 复盘就是反向传播
- **故事入口**:backprop 把最终的误差,沿着每一层分配责任。复盘做的是同一件事:把一个结果,翻译回"哪一步该改"。
- **机制**:credit assignment——没有复盘,经历只是前向传播一次就丢了;有复盘,经历才变成能更新权重的梯度。
- **落点**:复盘不是写总结,是把经历变成以后还能用的更新量。
- **来源**:E03。

## 4.3 找到能给你干净梯度的人
- **故事入口**:有人给你的反馈是高方差噪声("挺好的""有潜力"),有人给的是低方差、指向明确的梯度。后者稀缺,因为说真话有成本。
- **机制**:信号 vs 噪声;高质量反对者 = 低方差、方向对的梯度。真话还涉及场合、关系、谁承担后果。
- **落点**:夸你的人多,能指出你盲区的人少;升级反馈系统,比独自硬想更聪明。
- **来源**:J01 + J02 + J03 合并,并入 I08(真话的成本)、I09(借外部脑)、C07/D06(把 AI 和对的人当外部梯度源)。

# M5 · 学习率:你敢下多大的注

> 步长太大,loss 发散、直接 NaN;步长太小,卡着不动。下注大小,是个数值稳定性问题。

## 5.1 All in 不是勇敢,是数值发散
- **故事入口**:学习率调太高,loss 一步冲上天,整个 run 报废。All in 是一样的:不是看你敢不敢,是看你错了还能不能活。
- **机制**:发散 vs 收敛;Kelly 仓位——有正期望也可能因为下注过大而归零。本金小的时候尤其要先活着。
- **落点**:集中下注是风险结构,不是勇敢姿态;核心是错了以后还能不能继续跑。
- **来源**:H05,并入 H02(避免大错)、G03(钱少时先别 all in 在收益率上)。

## 5.2 小步快跑:用 mini-batch 买梯度
- **故事入口**:SGD 不是一次看完全部数据再走一步,而是用一个 batch 的带噪估计就走。小实验就是 mini-batch:先发 20 条内容、先服务 3 个客户、先做 MVP。
- **机制**:小步、多次、带噪;不是学会了才开始,是开始了才知道该学什么。
- **落点**:普通人最好的策略不是押中一个大机会,是持续做低成本小实验。
- **来源**:H03,并入 A05(边做边学)、H04(好风险可控可复盘)。

## 5.3 情绪稳定 = 梯度裁剪
- **故事入口**:一次异常大的梯度能直接毁掉训练好的权重,所以要 gradient clipping。情绪稳定就是给自己加裁剪:不让一次冲动更新掉整个模型。
- **机制**:gradient clipping;冲动辞职/合伙/投资/发言 = 一次没裁剪的巨大更新。
- **落点**:情绪稳定不是没情绪,是不让一次短期波动破坏长期积累。
- **来源**:J10。

# M6 · 噪声与探索:为什么需要随机

> SGD 的噪声不是缺陷。正是这点随机,让你跳出局部最优、泛化得更好。

## 6.1 噪声不是 bug:随机帮你跳出局部最优
- **故事入口**:full-batch 梯度下降经常陷在坑里,反而是 SGD 的 batch 噪声把模型抖出局部最优。生活里那些"计划外"的连接,作用一样。
- **机制**:stochasticity 帮助 escape local minima、改善泛化;过度计划 = full-batch,稳定但容易卡死。
- **落点**:认识新人、读杂书、试新工具——给系统留出意外连接的空间。
- **来源**:H07。

## 6.2 探索 vs 利用:好奇心是你的探索率
- **故事入口**:RL 里 ε 设成 0,智能体只会在已知的小路上打转,永远找不到更好的策略。好奇心就是把 ε 保持大于零。
- **机制**:exploration-exploitation tradeoff;有趣不是玩物丧志,是维持一个非零探索率,持续更新自己。
- **落点**:好奇心是一个人持续更新的发动机,也是长期生产力。
- **来源**:J08。

## 6.3 看不懂的不是机会,是分布外样本
- **故事入口**:模型在 OOD 输入上的预测毫无可信度,但置信度还可能很高。"别人暴富的故事"对你就是 OOD:你以为是机会感,其实是陌生感。
- **机制**:out-of-distribution / 高不确定性;陌生 ≠ 机会。
- **落点**:你看不懂的暴富,通常不是你的机会,是别人的叙事。
- **来源**:H06。

# M7 · 过拟合:在回测里赚翻,上线就亏【旗舰模块】

> 这是你最该独占的一块。别人只能讲段子,你能用真实的回测/训练数据讲清楚机制。建议整档节目从这里开始录。

## 7.1 过拟合:记住答案,不等于学会
- **故事入口**:一个在训练集上 99% 准、上线就亏钱的模型/策略。它没学到规律,只是背下了样本。
- **机制**:memorization vs generalization;努力没有积累 = 过拟合到一个一次性任务,换个场景就归零。
- **落点**:有些努力做完就结束(过拟合单一任务),有些会泛化到未来。
- **来源**:C05 的回测角度 + B03(努力不积累)。

## 7.2 幸存者偏差:回测里那个"他只买一只股票赚翻了"
- **故事入口**:用真实回测演示——只看活下来的样本,任何烂策略都能讲成神话。这是你能拿数据现场拆穿的一期。
- **机制**:survivorship bias / selection bias;成功故事是被幸存者筛选过的样本。
- **落点**:只买一只股票赚翻,可能是高认知,也可能只是幸存者叙事;区别在于错了还能不能活。
- **来源**:H05 的幸存者角度。

## 7.3 单一模型会过拟合;品味是泛化能力
- **故事入口**:单个模型再强也会在某些分布上崩,所以要 ensemble、要正则。只用一种思维模型看世界,是同样的过拟合。
- **机制**:ensemble / regularization;多个思维模型互相校正 = 降低方差;品味 = 在细微差别上不过拟合、能分辨真假克制油腻的判断力。
- **落点**:一个模型看世界是自信,多个模型看世界才接近清醒;生成变便宜后,品味变成商业能力。
- **来源**:C08 + F07 合并,并入 C06(知道什么不做 = 减法/正则)。

## 7.4 高质量勤奋 = 提升泛化,不是多刷几遍训练集
- **故事入口**:同一个小数据集多跑 100 个 epoch,只会过拟合得更狠。低质量勤奋就是这样:重复消耗,不进步。
- **机制**:more epochs on tiny data = overfit;高质量勤奋 = 换更好的数据、正则、反馈和环境,而不是单纯加量。
- **落点**:勤奋不是用来对抗天赋的,是用来把自己放进一个更好的系统里。
- **来源**:J07。

# M8 · 不归零:活下来才能收敛

> 收敛要很多步,但一次 NaN 就能让整个 run 作废。复利的前提是不归零。这是金工和 ML 共有的铁律。

## 8.1 活下来,是所有非线性回报的前提
- **故事入口**:训练到一半一次 NaN,前面几天的进度全没了,不管之前 loss 降得多漂亮。Ruin 是一道吸收壁——碰到一次就出局。
- **机制**:absorbing barrier / ruin;黑天鹅不是用来预测的,是用来防守的——保现金流、不过度负债、避免单点依赖、保留选择权(optionality)。
- **落点**:上行空间再大,如果一次失败让你出局,就不值得;面对黑天鹅,重点不是猜中,是别被一次清零。
- **来源**:H09 + H08 合并,并入 G09(选择权 = 期权式的凸性防守)。

## 8.2 复利不是快,是不归零
- **故事入口**:复利是连乘,中间出现一个零,前面所有项全废。所以慢一点没关系,归零一次就致命。
- **机制**:乘性过程对零敏感;滚雪球 = 长坡(足够时间)+ 湿雪(能粘住的积累)+ 不中断。
- **落点**:复利需要一个不轻易归零的容器:写作、代码、信誉、健康。
- **来源**:B06 + B07 合并。

## 8.3 资产 = 能替你继续训练的 checkpoint
- **故事入口**:一个存好的 checkpoint,你不在场它也在,随时能 resume、能 fine-tune。资产就是这种东西;负债是持续掉点、需要你不断喂算力维护的东西。
- **机制**:checkpoint vs 持续损耗;资产判断不看它多高级,看它未来会不会替你工作。
- **落点**:工资由公司发,但能力、作品、流程、信誉这些 checkpoint 要归自己。
- **来源**:B05 + G06 合并,并入 B03(没留下 checkpoint 的努力)。

## 8.4 身体和情绪,是你的算力本金
- **故事入口**:再好的模型,GPU 宕了也跑不动。Burnout 就是硬件故障;睡眠和运动是给算力本金充电。
- **机制**:compute budget / 硬件;所有长期计划都跑在这块本金上。
- **落点**:身体不是人生的附属品,是你所有长期计划的算力本金。
- **来源**:J09。

# M9 · 杠杆与规模化:从一次性脚本到系统

> 一个人的时间是线性的。代码、内容、资本、AI、流程、关系,都能让一次判断产生多次回报。规模化是非线性的来源。

## 9.1 杠杆总论:写一次,运行很多次
- **故事入口**:一个模型训练一次,之后服务上百万次推理。代码、内容、资本、关系都是这种"一次投入、多次结算"的放大器。
- **机制**:fixed cost 摊到无限次调用;代码最迷人的地方是你不在场时它还在工作。资本杠杆 = 放大已被验证的判断,不是替代判断。
- **落点**:非线性回报不靠更努力,靠让努力被杠杆放大。
- **来源**:D01 + D02 合并,并入 D05(资本杠杆)、D06(关系杠杆)。

## 9.2 从一次性脚本到可复现 pipeline
- **故事入口**:notebook 里跑通一次的 hack,和能稳定上线、别人也能复用的 pipeline,是两个东西。把经验从"只在你脑子里"变成 pipeline,才是资产化。
- **机制**:reproducible pipeline / SOP / 模板;Skill = 经验压缩包(目标、场景、步骤、判断标准、验收清单),不是一句 prompt。
- **落点**:一个人的临场发挥是线性的;一个流程才能被反复调用。好咨询卖的也是压缩过的判断,不是时间。
- **来源**:D04 + E01 + E02 合并,并入 E04(咨询 = 压缩判断)、F05(内容飞轮 = 内容生产 pipeline)、G07(买课不行动 = 下载了数据集却从不训练)。

## 9.3 个人能力有上限,系统才有非线性
- **故事入口**:单卡训练有天花板,分布式才能 scale。招人不是把活丢出去,是把判断和流程分片(shard)出去——讲不清目标和标准,招越多越累。
- **机制**:单机 vs 分布式;系统的上限 > 个人的上限。价值观 = 组织的 inductive bias / 正则项:没人盯着时的默认行为。
- **落点**:招人是转移判断,不是转移任务;AI 不是不招人的理由,是重新定义岗位的理由。
- **来源**:I07 + I05 合并,并入 I06(AI 与招人不互斥)、I01(工作是合作)、I02(价值观 = inductive bias)、I03(个人 vs 集体利益)、I04(早期团队一起长出来,刘邦作故事入口)。

## 9.4 Agent:让判断在你不在场时继续运行
- **故事入口**:你做 Agent 的本行——一个能在你睡觉时持续执行你的判断的系统。这是 AI 杠杆最具体的形态,也是只有你能讲透的一期。
- **机制**:agent / 自动化 = 把判断封装成可在无人值守时运行的流程;AI 最值钱的用法是陪你迭代问题,不只是替你给答案。
- **落点**:AI 不是替你成为神,是扩大你能调动的能力半径,并让判断离开你的在场也能工作。
- **来源**:D03 + C07 合并(全新成集,接你的 Agent 背景)。

# M10 · 部署:让模型上线、被看见

> 一个躺在 notebook 里的模型,不产生任何价值。内容就是部署——把一段思考放进时间、搜索、推荐和未来的场景里。

## 10.1 内容 = 把模型部署上线
- **故事入口**:模型不 serve 就等于不存在。内容(播客、文章)就是把脑子里的判断部署成对外服务、长期被调用的资产。
- **机制**:deployment / serving;播客是慢内容、长尾资产,说完不结束,会在时间里继续发酵、被引用、被切片再分发。
- **落点**:内容是认知的资产化,不是表达欲的残留物。
- **来源**:F01 + F04 合并,并入 F05(内容飞轮)。

## 10.2 命中不是爆款:优化 precision,不是曝光
- **故事入口**:你完全可以用 precision / recall 讲清楚自己"不搞爆款"的直觉:追曝光是在拉 recall(够大但脏),你要的是 precision——小播放量也能命中精准的人。
- **机制**:precision vs recall;爆款是高 recall 低 precision,标题党就是为了 recall 牺牲 precision。
- **落点**:内容真正的回报不是播放量,是在未来某个时刻命中了某个对的人;同理,代码能跑 ≠ 有人要(生产能力 ≠ 商业能力)。
- **来源**:F03,并入 E06(会写代码 ≠ 靠代码赚钱)。

## 10.3 一鱼多吃 = 数据增强,多采样几次命中
- **故事入口**:一个样本做增强,多生成几个视角,提高被正确识别的概率。一个好想法拆成播客、长文、短视频、金句,是同一回事。
- **机制**:data augmentation / 多次采样;不是偷懒复用,是增加一个好想法被世界命中的次数。
- **落点**:一鱼多吃的本质是提高一个判断被看见的概率。
- **来源**:F02。

## 10.4 形式是理解的入口:取名字与界面
- **故事入口**:用户第一次接触你的产品,看到的是名字、封面、文案——那是你这个"模型"对外暴露的 API。API 设计烂,再好的内核也没人用。
- **机制**:接口 / API 设计;名字影响记忆、理解和传播;App 只是外壳,工作流才是价值,但外壳决定了价值会不会被理解。
- **落点**:形式很多时候不是包装,是内容被理解的入口。
- **来源**:F06 + E05 合并。

# 2. 第一季录制顺序(12 期)

不按 M1→M10 平推,而是**先用你最能立住、最像你的一期开场**,再把世界观和机制铺开。原则:故事先行,机制随后,前两期就要让听众感到"这个工程师在拆我没拆过的东西"。

1. **过拟合:在回测里赚翻,上线就亏**(7.1)——最强、最独占,直接定调。
2. **幸存者偏差:那个"只买一只股票赚翻了"的人**(7.2)——接着用数据拆穿叙事,建立"我会用机制说话"的人设。
3. **为什么这档节目叫「非线性回报」**(序.1)——这时再上世界观,听众已经信你了。
4. **AI 让什么变便宜,让什么变贵**(序.2)——铺成本结构这条暗线。
5. **你在最小化哪个 loss**(1.1)——目标定义错,越努力越偏。
6. **人生是非凸优化,不是解方程**(2.1)——没有标准答案,只有提高概率。
7. **迷茫不是想太少,是没有梯度**(4.1)——反馈即学习,很抓人。
8. **All in 不是勇敢,是数值发散**(5.1)——学习率 / Kelly,金工主场。
9. **活下来,是所有非线性回报的前提**(8.1)——不归零这条铁律。
10. **从一次性脚本到可复现 pipeline**(9.2)——经验资产化,你的第一手素材。
11. **Agent:让判断在你不在场时继续运行**(9.4)——接你的 Agent 背景,AI-native。
12. **内容 = 把模型部署上线**(10.1)——收在"为什么做这档播客"上,自然闭环。
